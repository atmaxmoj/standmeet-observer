## Quality criteria

1. **Concise**: 4-7 steps max. If longer, split into separate routines
2. **No redundancy**: don't repeat in constraints what's already in steps
3. **Transferable**: no specific tool names, file names, or commands — describe the PATTERN
4. **Proactive + reactive**: include BOTH preventive setup AND failure recovery
5. **Mental model = one sentence**: the core insight, not a paragraph

## Structure

```
trigger:         When does this activate? (situation type, not specific event)
goal:            One sentence. What does success look like?
steps:           4-7 steps. At least one IF/THEN and one ON FAILURE.
failure_handling: One sentence. When to persist vs abandon.
constraints:     2-4 items. NEVER/ALWAYS rules that aren't already in steps.
tradeoff_policy: One sentence. What they protect vs sacrifice.
mental_model:    One sentence. The core causal insight.
```
