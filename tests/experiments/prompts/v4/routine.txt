## What separates a great routine from a mediocre one

MEDIOCRE routine:
```
Step 1: Identify the problem
Step 2: Research solutions
Step 3: Implement the solution
Step 4: Verify it works
```
This is useless — it describes what ANYONE would do.

GREAT routine (like Karpathy's program.md):
```
Step 1: Read the last 50 lines of the error log (not the first — root cause is at the bottom)
Step 2: IF the error is a missing dependency THEN pip install and retry
        IF the error is OOM THEN reduce batch size by half
        IF the error is NaN loss THEN revert last change and try lower learning rate
Step 3: Run for exactly 5 minutes. IF val_bpb improved, keep. IF not, git reset.
ON FAILURE: If you can't fix it in 2 attempts, abandon this idea entirely and move on.
```
This is actionable because it has SPECIFIC conditions, SPECIFIC actions, and SPECIFIC failure criteria.
